po-input directive takes care of the validation on the input field, its styling and binding the result to the model.

NOTE: when a non-default po-input type is provided, angular supported elements are not recommended. Instead, it is recommended to create a new type and contribute back here. See the contributing section.

Supported Arguments

  • *po-input="string" an enumeration of predefined input validations, defaults to "default"
  • *ng-model="string" (AJS) binds the validated input value to the provided model value
  • name="string" (AJS) access to the angular forms input result via $scope[formName][inputName]
  • po-label="string" sets the label. Angular evaluated (i.e. string value must use single quotes)
  • placeholder="string" (H5) sets the placeholder
  • class="string" (H5) classes specified here will be added in addition to the default .po-input class
  • ... angular supported input elements

Every po-input type supports default values for po-label and po-input. These can be overriden by implementing these attributes. All non-default po-input type examples below use the defaults. Taking the postcode as an example:

All po-inputs need to be wrapped in a form or ng-form element to work. This is the same for all angular form element components.

Default error handling is provided to input types, which will be triggered if a field returns invalid for regex patterns or ng-required. These default messages can be customised by passing in a custom error object for that failure type, or can be turned off by passing in an error message with empty strings. Examples for these behaviours are shown below, along with an alert panel to display the errors.

Postcode with default attributes:

//example controller.js - This overrides default ng-required message $scope.postCodeReqErrorObj = {type: 'Alert', message: 'Please enter a Postcode in this box [Custom override example]'};

Postcode with custom label and postcode:

//example controller.js //blank strings in an error object suppress the creation of an error message $scope.postCodeBlankPatternErrorObj = {type: '', message: ''};

Composite input types use the po-input above under the covers. Each directive is an element directive

Supported Arguments for all Composite inputs

  • <[input]>="element" this is the directive name for the composite input, e.g. po-address-manual-input
  • *ng-model="string" (AJS) binds the validated input value to the provided model value

All po-inputs need to be wrapped in a form or ng-form element to work. This is the same for all angular form element components.

Notice that the name, placeholder and po-input values are not supported for form composites. If you need to modify these, question the design and if it is valid, create a new composite type. See the contributing section.

po-select-input array usage

takes an array of select options as the list attribute. These are displayed when dropdown is toggled. When selected, the provided ngModel's view and model values are updated to the selected option. If a string is passed to the po-placeholder attribute, this string will be displayed as the initial placeholder otherwise the input will default to the first item in the list.

The options passed to the list attribute can also be live updated within the view, which can be simulated by clicking the below button.

Resulting ng-model

model.selectBoxModel = {{model.selectBoxModel}}
//example.controller.js 'use strict'; angular.module('poUiDocsApp') .controller('PoInputsCtrl', function ($scope) { $scope.selectBoxItems = ['Salmon', 'Trout']; $scope.selectBoxModel = ''; $scope.selectBoxLabel = 'Select Fishing Licence Type'; $scope.selectBoxPlaceholder = 'Select an option'; });

use attribute ng-required to determine whether or not to validate the selection of the drop down.

po-select-input model and view usage

takes an object with property names corresponding to desired modelValues and values corresponding to desired viewValues passed in the po-select-options attribute. View values are displayed when dropdown is toggled. When selected, the provided ngModel's view and model values are updated to the selected option's view and model value respectively.

The options passed to the po-select-options attribute can also be live updated within the view, which can be simulated by clicking the below button.

Resulting ng-model

selectBoxModel = {{model.poSelectInputSelectOptionsModel}}
//example.controller.js 'use strict'; angular.module('poUiDocsApp') .controller('PoInputsCtrl', function ($scope) { $scope.selectOptions = {modelVal1: 'some label', modelVal2: 'some other label'}; $scope.model.poSelectInputSelectOptionsModel = ''; $scope.selectBoxLabel = 'Select Fishing Licence Type'; $scope.selectBoxPlaceholder = 'Select an option'; });

Switch Input

The switch button turns a checkbox into an on/off switch that returns a boolean. Options that can be passed to this directive are label, which should be a variable containing the text to be shown next to the checkbox, and ng-model, which should be a boolean where true is checked. In the example below, switchValue is a variable on the scope that has the value TRUE

Resulting ng-model

switchValue = {{switchValue}}
angular.module('poUiDocsApp') .controller('PoInputsCtrl', function ($scope) { $scope.switchLabel = "Disabled License?"; $scope.switchValue = false; });

Toggle Button Select Input: Simple version

The Toggle Button Select Input allows users to select a value by clicking a button from a selection of options. The values for the buttons can be passed in as an array from a controller. The number of buttons displayed will be equal to the length of this array - 2 to 5 buttons are supported in the current version.

Resulting ng-model


toggleSelectedSimpleVersion =
angular.module('poUiDocsApp') .controller('PoInputsCtrl', function ($scope) { //Toggle label $scope.toggleButtonLabelSimpleVersion = 'Type of Licence'; //Model $scope.toggleSelectedSimpleVersion = ''; //List of items to be selected (simple version: array of strings) $scope.toggleButtonItems = ['Salmon & Sea Trout', 'Non-Migratory Trout & Coarse', 'Upgrade']; //Number of columns used to organize the toggle $scope.columnToggleSelectButtonSimpleVersion = 6; ... //How to reset the toggle select input $scope.toggleSelectedSimpleVersion = ''; ... //How to programmatically set the toggle button $scope.toggleSelectedSimpleVersion = 'Salmon & Sea Trout'; });

Toggle Button Select Input: Advanced version

The Advanced Toggle Button Select Input allows users to select a value by clicking a button from a selection of options, as well as by custom name attribute. For the example below, in the toggleSelectedAdvancedVersion box, if you type any of the model names that were setup in the controller (see $scope.toggleButtonItemsOpts below) into this box, you will see that the corresponding button is selected.

Resulting ng-model


toggleSelectedAdvancedVersion =
angular.module('poUiDocsApp') .controller('PoInputsCtrl', function ($scope) { //Toggle label $scope.toggleButtonLabelAdvancedVersion = 'Type of Licence'; //Model $scope.toggleSelectedAdvancedVersion = ''; //List of items to be selected (advanced version: array of objects with model and view attributes) $scope.toggleButtonItemsOpts = [{view:'Salmon & Sea Trout',model:'salmon'}, {view:'Non-Migratory Trout & Coarse',model:'trout'}, {view:'Upgrade',model:'upgrade'}]; //Number of columns used to organize the toggle $scope.columnToggleSelectButtonAdvancedVersion = 6; ... //How to reset the toogle select input $scope.toggleSelectedAdvancedVersion = ''; ... //How to to programmatically set the toggle button $scope.toggleSelectedAdvancedVersion = 'salmon'; // pay attention to its type, should match the model type });

Manual Date Input

The manual date input field will allow users to add dates in text form. It will validate dates on blur to prevent invalid date types (e.g. 41 could not be typed in the day field).

Be aware a model date must always be provided as a parameter in a valid date format.


Resulting ng-model

manualDateValue = {{manualDateValue}}
angular.module('poUiDocsApp') .controller('PoInputsCtrl', function ($scope) { $scope.manualDateLabel = 'Date of Birth'; $scope.manualDateDisabled = false; $scope.maxDate = new Date('2018/3/17'); $scope.minDate = new Date('2000/3/17'); $scope.minDateErrorObj = {type: 'Alert', message: 'Please choose a more recent date'}; $scope.maxDateErrorObj = {type: 'Alert', message: 'Please choose an older date'}; $scope.manualDateValue = new Date('2013/3/17'); });

Date Picker

A clean, flexible, and fully customizable date picker. User can navigate through months. The datepicker shows dates that come from other than the main month being displayed. These other dates are also selectable.

Defaults to empty value but a default value can be easily set (see below). Use "min" and "max" attributes to set min / max allowed date (type: Date)
