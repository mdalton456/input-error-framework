The input error framework was developed to allow other project developers to be able to easily add front-end error handling to their user stories.

It is designed to be developer-friendly yet easily customisable. Developers can either use the default validation types, or pass their own custom errors into a directive. Any errors or info messages will collect on a modal/alert panel on the page, which can be used to prevent progress from the page if required.

Validation types are set using po-input. All po-inputs need to be wrapped in a form or ng-form element to work.

Default error handling is provided to input types (e.g. Email address, phone, currency, time etc.), which will be triggered if a field returns invalid for regex patterns or ng-required. These default messages can be customised by passing in a custom error object for that failure type, or can be turned off by passing in an error message with empty strings. Examples for these behaviours are shown below, along with an alert panel to display the errors.

Postcode with default attributes:


//example controller.js - This overrides default ng-required message $scope.postCodeReqErrorObj = {type: 'Alert', message: 'Please enter a Postcode in this box [Custom override example]'};

Postcode with custom label and postcode:


//example controller.js //blank strings in an error object suppress the creation of an error message $scope.postCodeBlankPatternErrorObj = {type: '', message: ''};
